# 修改说明（2026-03-19）：辅助损失稳定性增强

## 背景

在 `slake-cp` 上使用 Full 配置（SupCon + DDC + ECC）训练时，出现大量：

- `[warn] non-finite gradients detected. skip optimizer step.`

导致优化步骤频繁被跳过，训练无法有效收敛。

## 修改目标

1. 降低 DDC 数值放大风险（分母过小与指数项放大）。
2. 对辅助损失总量增加可配置上限，避免单 batch 梯度异常峰值。
3. 保持现有训练流程不变，提供命令行可控参数。

## 代码改动

### 1) 新增命令行参数

文件：`main_arcface.py`

- 新增 `--max-aux-loss`（默认 `5.0`）
- 含义：对“已乘权重后的辅助损失贡献”做上限裁剪，提升训练稳定性。

### 2) 训练损失计算稳定化

文件：`train_arcface.py`

- 新增常量：
  - `KL_DENOM_EPS = 1e-4`
  - `EXP_CLAMP = 10.0`（由 20.0 下调）
- 主要调整：
  1. 将 SupCon/DDC/ECC 先累加为 `aux_total`。
  2. 统一计算 `scaled_aux = aux_scale * aux_total`。
  3. 若 `max_aux_loss > 0`，执行 `scaled_aux = scaled_aux.clamp(max=max_aux_loss)`。
  4. 最终再将 `scaled_aux` 加回主损失。
- DDC 细节：
  - `kl2_safe = kl2.clamp_min(KL_DENOM_EPS)`，减少分母过小导致的不稳定。
  - 降低指数项裁剪范围，减弱梯度放大。

## 使用建议

### 推荐稳定训练命令（Full 配置）

```bash
python main_arcface.py \
  --name f_nowp_a1_e100_safe \
  --gpu 0 \
  --dataset slake-cp \
  --aux-warmup-epochs 10 \
  --aux-loss-weight 1.0 \
  --max-aux-loss 4.0 \
  --epochs 100 \
  --seed 1
```

### 若仍有较多溢出告警

可进一步收紧：

- `--max-aux-loss 3.0` 或 `2.0`
- 结合更长 warmup（例如 `--aux-warmup-epochs 15`）

## 预期效果

1. 减少 non-finite 梯度告警频率。
2. 降低 `skip optimizer step` 的占比。
3. 在保持 Full 训练策略的同时提升收敛稳定性。
